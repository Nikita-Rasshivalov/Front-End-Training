1)Поскольку ваш URL-адрес является относительным (это просто "...json"), 
он сопоставлен с URL-адресом страницы, на которой выполняется код. В вашем случае это выглядит 
как file: /// something - то есть файл, который вы загружены прямо из вашей файловой системы,
 а не по запросу с сервера. Chrome не позволяет извлекать из файловой схемы. 
Происхождение файловой схемы равно нулю. Команда Chrome интерпретирует политику одного и того
 же происхождения в том, что ни один источник, даже он сам, не должен соответствовать нулю. 
(На мой взгляд, разумная интерпретация, но мнения могут быть разными.)

При веб-разработке вы хотите работать через серверный процесс, потому что страницы, 
загруженные непосредственно из файловой системы, ведут себя по-разному в некоторых, иногда тонких отношениях,
по сравнению со страницами, загруженными с серверов.

Это можно сделать двумя способами:

Используйте возможности своей IDE, если вы их используете, и
 / или расширения к ней. Например, для VSCode существует расширение «живого сервера»,
 которое позволяет очень легко запускать ваш код с минимального сервера.
Используйте реальный процесс веб-сервера, работающий локально. Есть простые, которые легко установить.
Используйте один из различных инструментов создания шаблонов для быстрого старта, которые настраивают
 для вас простые проекты, часто через npm (и Node.js), с командой, которую вы можете использовать для 
создания и запуска проекта локально в режиме разработки.

2) Обработка запроса на изменение была выполнена на стороне сервера ( на node-js). Для запуска сервера необходимо перейти из
корня в папку server (cd server) и запустить index.js и прописать (npm run dev). После запуска сервера переходим на наш 
стартовый сервер(visual studio live server) и производим редактирование в пятом поле.
3) Так как у нас в роли сервера выступают файлы то при любом изменении данных файла будет происходить перезагрузка страницы 
веб-браузера. Если же у нас реальный сервер то при редактировании данных файла, страница перезагружаться не будет.
Примечание:веб-браузеры не позволяют через запросы редактировать файлы из за политики безопасности(на стороне клиента).
